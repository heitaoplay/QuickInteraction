## 1. 总体结论

**功能层面是过关的。** 这个脚本经过 v0.5.4~v0.5.9 多轮真机验证，动作发包、ECHO/LSCG 兼容、自己模式、全员广播、组合执行都已跑通，作者还做了 `ServerSend → ActivityRun` 降级、`FocusGroup` 补位、防重复加载等工程细节，说明作者对业务理解深、有交付意识。这是团队里难得能打硬仗的代码。

**但工程层面存在系统性欠债，会随版本继续累加：** 单文件 2261 行、22 处调试日志混在生产代码（一期已剥离）、多处 `catch(_){}` 静默吞错、标签/过滤逻辑四函数交织重复。其中「状态散落 23 个全局变量」已于二期归并为单一 `state` 对象。剩余问题单个看不致命，叠加起来就是"新人读不懂、改一处怕炸一片、bug 查不到源头"——这正是团队技术能力上不去的根因。

**优先级判断：** 当前最该先治的是「错误可见性」和「调试日志未分离」两条，它们直接拖慢每一次排障；其次是「状态/逻辑归一」和「工程化基建」。

## 3. 问题清单（按优先级）

### P0 — 阻断级 / 必须立即处理

**P0-1 静默吞异常，配置与数据可能无声丢失**
- 位置：`loadStorage`/`saveStorage` L168–172；`getActionsForPart` L197–199、L463、L540；`main` L2170、L2177、L2182–2187。
- 现象：至少 9 处 `catch(_){}` 或 `catch(e){ console.warn }` 后不处理。
- 影响：localStorage 读写失败、动作预校验失败、mod 注册回退失败——全部静默。用户配置静默丢失、功能悄悄变弱，排障时毫无线索。
- 修复：禁止空 catch。存储层失败应 `console.error` + 至少 toast 一次；预校验失败应明确告知「该动作当前不可用」而非默默跳过。

**P0-2 调试日志混在生产代码，每次点击都打**
- 位置：`makeActivityPacket` L428、L430（每次构造包都 `console.log` ActivityAsset 详情）；`main` L2148、L2161、L2179、L2238 等共 22 处。
- 影响：① 性能——动作高频触发时控制台被刷屏；② 体验——普通用户打开 F12 看到一堆内部日志；③ 维护——发布版本与调试版本无区分。
- 修复：引入 `DEBUG` 常量 + `logD()` 封装，发布构建剥离 `console.log`；仅保留 `console.warn/error` 用于真实异常。

**P0-3 无工程化基建，脚本满天飞**
- 位置：工作区根目录 40+ 个 `check_*.js` / `diag_*.js` / `test_*.js` / `verify_*.js` / `inject_*.js`。
- 影响：无法区分「交付物」与「临时探针」；新人不知道哪个能跑、哪个已废弃；协作必乱。
- 修复：建 `src/`（交付源码）、`dev/`（调试探针，不进发布）、`tests/`；引入轻量 lint（ESLint）+ 一个打包/构建步骤（哪怕只是合并+strip 调试日志）。

### P1 — 重要 / 本版本内应重构

**P1-1 状态管理散乱，toggle 模式重复三份**
- 位置：23 个全局 `let` L42–62；`updateAllButtonVisual` L657 / `updateFavButtonVisual` L674 / `updateSelfButtonVisual` L719 三个近同函数。
- 影响：状态边界不清，新增开关要复制一套变量+视觉函数，易漏。
- 修复：✅ 已完成前半——23 个散落全局变量已归并为单一 `state` 对象（v0.6.0），225 处引用统一改写。toggle 视觉函数统一（`setMode` + `renderToggle`）留待后续小改。

**P1-2 标签/过滤四函数职责重叠、判断重复**
- 位置：`getActivityLabel` L243、`getActivityLabelFallback` L251、`hasActivityLabel` L276、`shouldKeepAction` L292。
- 现象：`'[STRING_RETRIEVAL_FAILED]'` 出现在 L234/L260/L281；`'MISSING'` 出现在 L233/L261/L281/L294；`'XSAct_'` 前缀判断重复 3 处。
- 影响：改一处过滤规则要同步改多处，极易漏改导致「一会儿显示乱码、一会儿动作被误删」（历史上 v0.5.7/0.5.8 正是这类补丁）。
- 修复：抽成单一 `ActionLabel` 模块：`normalize(name)` / `lookupLabel(name, group)` / `shouldShow(name, group)`，内部集中常量与判断。

**P1-3 版本号双维护**
- 位置：`// @version 0.5.9` L5 与 `const VERSION='0.5.9'` L28。
- 影响：改一漏一，用户看到的版本与代码内版本不一致。
- 修复：单一版本源，构建时注入；或 `@version` 直接读 `VERSION` 常量。

### P2 — 优化 / 技术债清理

**P2-1 魔法字符串/数字散落**
- 颜色 `'#FF5C7A'`(主题)、`'#FF5C5C'`(错)、`'#888'` 多处；`delay` 120/160；Content 前缀 `'ChatSelf-'`/`'ChatOther-'`/`'Label-ChatOther-'` 重复拼接。
- 修复：抽到 `THEME` / `CONFIG` 常量对象。

**P2-2 executeAction 三层 try 嵌套**
- 位置：外层 L451 → 预校验 L456 → ServerSend L474–483(finally)。
- 修复：拆出 `validateAction()` 和 `sendPacket()`，主流程线性调用，finally 仅用于还原 `FocusGroup`。

**P2-3 dead code**
- 位置：`presets` L54 注释「预留，尚未实现 UI」。
- 修复：要么实现，要么删除声明，不留「半成品占位」。
