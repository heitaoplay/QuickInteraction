## 一、版本与日期

- **版本号**：v1.1.0
- **发布日期**：2026-07-21

## 二、主要更新内容

- **「我的动作」列表交互重设计**
  - 移除原勾选框，改为**点击卡片高亮多选**。
  - 新增**「编辑模式」**：支持批量关闭 / 批量删除、全选 / 取消全选、拖拽排序。
- **拖拽排序持久化**：编辑模式下拖动自定义动作调整顺序，结果写入账号 / 本地，**刷新或重载后顺序保留**。
- **一键开关全部**：工具栏新增按钮，一键将所有自定义动作**全部开启或全部关闭**，状态即时同步。
- **工具栏图标化**：「编辑模式」「一键开关」改为图标按钮（无文字、避免溢出）；编辑模式下搜索框占位保留，**按钮组位置不跳动**。
- **「我的动作」标签图标更新**：更换为统一风格的糖果 / 魔法棒 SVG，激活态配色与其他标签一致。
- **预览变量修复**：修复「对自己时显示」预览中「对方」变量误解析为玩家自己的问题；预览现按作用域正确替换 `{TargetCharacter}`。
- **占位符区块优化**：编辑器「可用占位符」区块移到「对他人时显示」与「对自己时显示」之间，按钮改为带绿（自己）/ 粉（对方）圆点的精致药丸按钮，增加悬停 / 点击反馈。

## 三、重要变更说明：书签安装方式修复

不少用户反馈「**书签安装方式失效**」。本次专门排查并修复：

- **排查结论**：书签安装链接本身一直是**有效**的。经实测，`https://heitaoplay.github.io/QuickInteraction/assets/main.js` 返回 `HTTP 200`、MIME 为 `application/javascript`、无 `X-Content-Type-Options: nosniff`、CORS 开放，浏览器可正常以 `<script>` 方式执行；脚本也不依赖任何油猴专有 API，且会等待 BC 的 `bcModSdk` 就绪后再注册。
- **真正的失效原因**：旧书签代码是「**静默注入**」——点书签后若你**还没进入已登录的聊天室**，脚本不会初始化入口（右下角不出现闪电图标），而书签本身**没有任何提示**，看起来就像「装了没反应 / 失效」。
- **本次修复**：
  1. 新书签代码增加**加载反馈**——成功 popup「已加载 ✓」，失败或不在聊天室则明确提示原因；
  2. 文档明确前提：**先进入 BC 聊天室并登录，再点书签**；每次刷新需重新点一次。

> 控制台安装方式同样存在「静默」问题，本次一并补充了同样的提示与前提说明。

## 四、升级提示与注意事项

- **油猴用户**：刷新 BC 页面即自动更新；脚本端内置 5 分钟轮询也会弹出更新提示，无需手动操作。
- **书签 / 控制台用户**：请按文档**重新创建书签**（复制新代码，注意保留开头的 `javascript:` 前缀）。
- **重复加载提醒**：若之前装过完整版 `quick-interaction.user.js`，建议先在油猴里删除，避免重复加载。
- **Beta 说明**：「我的动作」功能当前仍为**测试版 (Beta)**，仍在打磨中，欢迎继续反馈。
